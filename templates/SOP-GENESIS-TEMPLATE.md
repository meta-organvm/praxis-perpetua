# SOP: Genesis Template (SPEC-023)

## 0. Metadata (The Contract)
- **objective**: [State the clear outcome intended]
- **input_conditions**: [Pre-conditions required for execution]
- **output_artifacts**: [The primary artifacts generated (e.g. PDF, Git Commit)]
- **command_sequence**: [The exact ordered list of commands to run]
- **quality_gates**: [The standards that must pass for artifact acceptance]
- **evidence_required**: [The logs, records, or proofs required for completion]
- **metrics_captured**: [The telemetry points to record (e.g. time, retries)]
- **revision_triggers**: [When this SOP must be evolved (e.g. >10% defect rate)]
- **owner**: [The human or agent responsible for the genesis of this SOP]
- **last_validated**: [YYYY-MM-DD of the last successful full execution]

## 1. Phased Execution

### 1.1 Genesis Phase
- Define the **Objective** and **Artifacts**.
- Validate **Input Conditions**.

### 1.2 Specification Phase
- Declare the **Command Sequence**.
- Establish **Quality Gates** and **Evidence** requirements.

### 1.3 Execution Phase
- Run the sequence.
- Capture step-level logging.

### 1.4 Verification Phase
- Validate Artifact against Quality Gates.
- Verify Evidence set is complete.

### 1.5 Telemetry & Persistence Phase
- Record captured Metrics.
- Link Evidence to the Artifact.

### 1.6 Revision Loop Phase
- If any gate fails or metrics show drift, trigger **Forced Revision**.

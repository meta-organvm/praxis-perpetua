---
name: Shell config philosophy
description: User wants shell config to be invisible, fast, and correct — zero maintenance, zero thinking about internals
type: user
---

User is not a shell/terminal enthusiast. They want their shell environment to be:
- Invisible — just works, never needs attention
- Fast — no perceptible delay on open
- Silent — no errors, warnings, or prompts
- Low-maintenance — no manual refresh, no "run this in your terminal first"

They don't enjoy configuring shell internals. Think "lazy king" — the system should serve them perfectly without ever requiring them to understand or touch it. When something breaks, it should self-heal, not ask for intervention.

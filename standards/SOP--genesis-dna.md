# SOP: Genesis DNA — Project Inception Bundle

## 1. Ontological Purpose

This SOP defines the minimum viable structure that every new repository in the ORGANVM system must have at birth. It is the "building block DNA" — the set of foundational skills and artifacts that fire at project creation, before any code is written.

Genesis is not onboarding (which assumes a repo exists). Genesis is the act of bringing a repo into existence with the correct structural genome so that all downstream SOPs (promotion, deployment, publishing) have the substrate they need.

## 2. Minimum Viable Repo Structure

Every new repo MUST contain these files after genesis:

1. `README.md` — following github-repository-standards (World-Class README pattern)
2. `LICENSE` — appropriate to the organ (MIT for III, CC-BY for V, etc.)
3. `.gitignore` — language-appropriate, generated from templates
4. `seed.yaml` — declaring organ membership, tier, produces/consumes edges
5. `CLAUDE.md` — agent context file (auto-generated by `organvm context sync`)

## 3. Procedure

### Step 1: Create Repository

Use `SOP--repo-onboarding-and-habitat-creation` for the full onboarding flow. This SOP governs the structural invariants that onboarding must satisfy.

### Step 2: Apply Repository Standards

Invoke the `github-repository-standards` skill:
- Branch protection on `main`
- Issue templates (if applicable)
- Community health files (CONTRIBUTING.md, CODE_OF_CONDUCT.md) — inherited from org `.github` repo

### Step 3: Register in System

1. Add entry to `registry-v2.json` with status `LOCAL`
2. Create `seed.yaml` with organ, tier, and edge declarations
3. Run `organvm registry validate` to confirm schema compliance

### Step 4: Generate Context Files

```bash
organvm context sync --organ <ORGAN_KEY>
```

This generates CLAUDE.md, GEMINI.md, and AGENTS.md with system context.

## 4. Output Artifacts

- Repository with 5 minimum viable files
- Registry entry (status: LOCAL)
- seed.yaml with valid edge declarations
- Auto-generated context files

## 5. Linked Skills

| Skill | Role in Genesis |
|-------|----------------|
| `github-repository-standards` | README structure, branch protection, health files |
| `github-repo-curator` | Repository organization and presentation |
| `github-profile-architect` | Org-level profile consistency |

## 6. Relationship to Lifecycle

Genesis is phase 0 — it precedes the promotion state machine entirely. A repo that has completed genesis enters the state machine at `LOCAL` and begins the `foundation` phase.

```
[genesis] → LOCAL (foundation) → CANDIDATE (hardening) → PUBLIC_PROCESS (graduation) → GRADUATED (sustaining)
```
